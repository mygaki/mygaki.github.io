# kubernetes

概述

kubernetes 简称K8S，其中8代表了”ubernete”的8个字符，是一个开源的自动化容器编排引擎

  • 是谷歌在2014年开源的容器化集群管理系统（go语音开发的）

  • 使用K8S进行容器化应用部署

  • 使用K8S利于应用扩展

  • K8S目标实施让部署容器化更加简洁和高效

特性

  • 自动装箱（自动部署）

  • 自动修复（自愈能力，监控检查）

  • 水平扩展

  • 服务发现（负载均衡）

  • 滚动更新

  • 版本回退

  • 密钥和配置管理

  • 存储编排

  • 批处理（一次性和定时任务）

K8S集群架构组件

Master(主控节点) 和 worker node(工作节点)

  1. master组件（集群控制节点，对集群进行调度管理，接收集群外用户对集群的操作请求）

    • ApiServer
      • 集群统一入口，以restful方式，交给etcd[^1]存储
    • scheduler
      • 节点调度，选择node节点应用部署
    • controller-manager
      • 处理集群中常见后台任务，一个资源对应一个控制器
    • etcd
      • 存储系统，保存集群相关数据

  2. worker node

    • kubelet
      • 类比master派给node节点的代表，负责管理节点中的容器
    • kube-proxy
      • 提供网络代理，实现负载均衡等操作
    • pod(Container Runtime)

K8S核心概念

  1. Pod

    • 最小组成单元
    • 一组容器的集合
    • 共享网络
    • 生命周期是短暂的

  2. Controller

    • 确保预期的Pod的副本数量
    • 无状态和有状态应用部署
    • 确保所有的node运行同一个Pod
    • 一次性任务和定时任务

  3. Service

    • 定义一组Pod的访问规则

由Serivce对外创建访问规则，由Controller创建Pod并运行

搭建K8S环境平台规划

  • 单master集群部署
  • 多master集群部署

部署kubunetes集群有多种方式：

  1. minikube

[^1]: etcd 是一个高可用的 Key/Value 存储系统，主要用于分享配置和服务发现
